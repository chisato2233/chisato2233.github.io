# 快速傅里叶变换

【注】菜鸡刚从 b 站大佬搬运的 FFT 科普里出来，所以写的东西可能会 “似曾相识”（真的有人看到这句话吗）

# 前言

# 提出问题

​ 现在我们想要处理这样的问题：给定两个多项式 A (x) 和 B (x), 求这样的 C (x) 满足：

A(x)×B(x)A(x)\times B(x) A(x)×B(x)

​ 很简单的问题，就和题面一样直白。这里的 A (x) 和 B (x) 都是已知多项式，我们只需要用中学生都学会的乘法分配律就可以轻松解决。

​ 然而，这样处理的问题在于，当 A (x) 和 B (x) 的阶数非常大的时候，这种算法无疑会浪费很多时间。让我们换一种思路解决这样的问题。

# 多项式的点表示法

​ 对于一个 n-1 阶多项式，我们只需要 n 阶个不同点就可以精确的给出，就像两点确定一条直线，三点确定一条抛物线一般。这个结论的证明非常简单易懂，你只需要证明下面这个线性方程有解即可

（P (x) 是系数为p0,p1,p2,⋯ ,pn−1p_0,p_1,p_2,\cdots,p_{n-1}p0​,p1​,p2​,⋯,pn−1​ 的多项式，找到了 n 个点x0,x1,x2,⋯ ,xn−1x_0,x_1,x_2,\cdots,x_{n-1}x0​,x1​,x2​,⋯,xn−1​）

[P(x0)P(x1)P(x2)⋯p(x2)]=[1x0x02…x0n−11x1x12…x1n−11x2x22…x2n−1⋯⋯⋯⋯⋯1xn−1xn−12…xn−1n−1][p0p1p2⋯pn−1]\left[\begin{matrix}P(x_0)\\P(x_1)\\P(x_2)\\\cdots \\p(x_2)\end{matrix} \right]=\left[\begin{matrix}1 & x_0 &x_0^2 &\ldots & x_0^{n-1}\\1 & x_1 &x_1^2 &\ldots & x_1^{n-1}\\1 & x_2 &x_2^2 &\ldots & x_2^{n-1}\\\cdots& \cdots&\cdots&\cdots&\cdots\\1 & x_{n-1} &x_{n-1}^2 &\ldots & x_{n-1}^{n-1}\\\end{matrix} \right]\left[\begin{matrix}p_0\\p_1\\p_2\\\cdots \\p_{n-1}\end{matrix} \right] ⎣⎢⎢⎢⎢⎢⎡​P(x0​)P(x1​)P(x2​)⋯p(x2​)​⎦⎥⎥⎥⎥⎥⎤​=⎣⎢⎢⎢⎢⎢⎡​111⋯1​x0​x1​x2​⋯xn−1​​x02​x12​x22​⋯xn−12​​………⋯…​x0n−1​x1n−1​x2n−1​⋯xn−1n−1​​⎦⎥⎥⎥⎥⎥⎤​⎣⎢⎢⎢⎢⎢⎡​p0​p1​p2​⋯pn−1​​⎦⎥⎥⎥⎥⎥⎤​

​ 系数矩阵是一个范德蒙德矩阵，他很显然是一个满秩矩阵 —— 只要保证任意两个 x 不相等，这说明上述的线性方程组有唯一解，定理得证。

​ 这代表着我们不需要做繁琐的多项式展开，只需在 A (x) 和 B (x) 上找到 n 个互不相等的点，将这些点的纵坐标相乘，获得表达 C (x) 的这一组点，最后通过求解上面的线性方程组，就能得到 C (x) 的具体而又精确的表达式。

​ 这真的足够简便吗？并非如此。你会发现当你试图进行第一步：找这样的点的时候，你将面临一个复杂的计算。这样的计算还要重复（n+1）次，从时间复杂的上来看，你似乎在南辕北辙。

# 正负对取点

​ 简单的随意去点似乎不能让我们事半功倍，但这正是 FFT 算法的精妙之一。FFT 用到的取点方法能够实打实地降低计算的复杂程度。

​ 假设P(x)=x3P(x)=x^{3}P(x)=x3, 你现在想取四个点，不妨就令x1=1,x2=−1,x3=2,x4=−2x_{1}=1,x_{2}=-1,x_{3}=2,x_{4}=-2x1​=1,x2​=−1,x3​=2,x4​=−2, 这样当你把 1 或者 2 带入计算的同时，你也得知了 - 1 和 - 2 所对应的函数值.

​ 根据函数的奇偶性可以 “咔！” 的一下将工作量减半。这是个投机倒把的小技巧，似乎只能在这样特别好看的函数上才能落实到位…
